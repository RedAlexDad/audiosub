# Project Instructions for OpenCode

## Commit workflow

Перед каждым коммитом необходимо:

1. Создать отчёт в `reports/` о том, что было сделано:
   - Что добавили
   - Что изменили
   - Какие были проблемы
   - Как их решили

Это делает pre-commit hook автоматически, создавая файл `reports/<дата_время>.md`.

2. Отредактировать созданный отчёт — заполнить содержимое.

3. Затем выполнить коммит.

## Commit format

Коммиты только на русском языке, формат:

```
<type>(<опциональная область>): <описание>
```

Типы: feat, fix, docs, style, refactor, test, chore
