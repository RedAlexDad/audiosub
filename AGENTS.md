# Project Instructions for OpenCode

## Commit workflow

Перед каждым коммитом необходимо:

1. Создать вручную заполненный отчёт в `reports/<дата_время>.md`:
   - Что добавили
   - Что изменили
   - Какие были проблемы
   - Как их решили

2. Добавить отчёт в индекс вместе со всеми изменёнными файлами.

3. Выполнить коммит (один раз, без amend).

Никаких автоматических хуков — отчёт создаётся руками и коммитится вместе с кодом за один раз.

## Commit format

Коммиты только на русском языке, формат:

```
<type>(<опциональная область>): <описание>
```

Типы: feat, fix, docs, style, refactor, test, chore
