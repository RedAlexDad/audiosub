# Commit Review

Скилл для подготовки коммита: ревью кода + составление отчёта.

## Когда использовать

Загружай этот скилл, когда пользователь:
- просит проверить код перед коммитом
- просит составить отчёт о проделанной работе
- говорит "подготовь к коммиту", "сделай ревью", "напиши отчёт"

## Процесс работы

### Шаг 1: Посмотреть изменения

Выполни `git diff --cached` чтобы увидеть уже проиндексированные файлы.
Если изменений нет — выполни `git --no-pager diff` чтобы увидеть
непроиндексированные изменения. Также посмотри `git --no-pager status`.

### Шаг 2: Понять контекст

Посмотри, какие файлы изменены. Если нужно — прочитай ключевые изменения
в файлах, чтобы понять суть работы.

### Шаг 3: Ревью кода (codereview)

Проанализируй изменения и напиши ревью по следующим критериям:

- **Логика**: нет ли багов, правильно ли работают изменения
- **Безопасность**: нет ли уязвимостей (особенно если это Rust с unsafe)
- **Стиль**: соответствует ли код стилю проекта (проверь `rustfmt.toml` если есть)
- **Избыточность**: нет ли дублирования, мёртвого кода
- **Консистентность**: изменения согласуются с окружающим кодом

Если находишь проблемы — сообщи о них пользователю и предложи исправить.
Не исправляй самостоятельно без согласия пользователя.

### Шаг 4: Составить отчёт

Создай файл `reports/<дата_время>.md`, где `<дата_время>` — текущая
дата и время в формате `YYYY-MM-DD_HHMMSS` (например, `2026-05-29_235959`).

Шаблон отчёта:

```markdown
# Report <дата и время UTC>

## What was added
- Перечисли новые файлы, новые функции, новые фичи

## What was changed
- Перечисли изменения в существующем коде

## Problems encountered
- Какие проблемы/ошибки встретились в процессе

## How they were solved
- Как эти проблемы были решены
```

Заполни каждый раздел на основе анализа изменений.
Если информации недостаточно — задай пользователю уточняющие вопросы.

### Шаг 5: Проверить формат коммита

Напомни пользователю, что коммит должен быть по формату:

```
<type>(<опциональная область>): <описание>
```

Где type: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`.

Коммит на русском языке.

## Важные замечания

- Отчёт создаётся, но **не индексируется и не коммитится** автоматически — пользователь делает это сам.
