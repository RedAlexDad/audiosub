# Report 2026-05-31 19:46 UTC

## What was added

- `src/tui/input.rs` — обработка MouseEvent: drag для изменения высоты панели partial-текста
- `src/tui/app.rs` — поля `partial_height` и `drag_start` для ресайза панели мышью
- `src/tui/capture.rs` — `EnableMouseCapture` / `DisableMouseCapture` при входе/выходе из TUI
- Константа `read_step = chunk_size / 4` — уменьшенный шаг чтения аудио (~25ms вместо ~100ms) для отзывчивости UI

## What was changed

- `src/tui/input.rs` — рефакторинг `handle_input`: разделён комбинированный `poll + let-else` на явный `poll → read → match`; теперь корректно обрабатываются MouseEvent-ы и KeyEvent-ы через единый `match`
- `src/tui/capture.rs` — добавлен `break` после `handle_input`, если `!app.running`: выход по q/Esc/Ctrl+D происходит немедленно, без ожидания audio-блокировки
- `src/tui/view/recognition.rs` — высота partial-панели берётся из `app.partial_height` вместо хардкода `3`, с клиппингом по доступной высоте
- `src/tui/app.rs` — улучшена дедупликация `partial_history`: новая строка не добавляется, если она является префиксом/расширением предыдущей (устранён спам почти одинаковыми строками)

## Problems encountered

- **Лаг/прерывистость VU-метра**: `capture.read(read_step)` возвращает `None` большую часть времени, потому что ресемплер (`rubato::Fft` с chunk_size=2048) копит входные сэмплы до `input_needed ≈ 1024` сэмплов. Из-за этого `update_audio_levels` вызывается только каждый 3-й проход цикла, VOL-метр обновляется рывками.
- **PulseCapture не Send**: `*mut pa_simple` внутри `libpulse_simple_binding::Simple` — raw pointer, Раст считает структуру `!Send`. Для многопоточности потребуется `unsafe impl Send`.

## How they were solved

- Уменьшение `read_step` с `chunk_size` (1600 сэмплов = ~100ms блокировки) до `chunk_size / 4` (400 сэмплов = ~25ms) дало частичное улучшение — цикл проверяет `app.running` и обрабатывает ввод в 4 раза чаще, но VOL всё ещё прерывист.
- `break` после `handle_input` решил проблему тормозного выхода — quit теперь мгновенный.

## Next steps (architectural)

Обсуждено и запланировано на следующий коммит — многопоточная архитектура:

```
[Capture Thread] ──raw f32──→ [ASR Thread] ──UiUpdate──→ [TUI Thread]
```

- **3 потока**: Capture (только pa.read), ASR (ресемплинг + engine), TUI (poll input + draw)
- **2 канала**: `mpsc::channel<Vec<f32>>` (Capture→ASR), `mpsc::channel<UiUpdate>` (ASR→TUI)
- **Stop**: `Arc<AtomicBool>` для сигнала остановки всем потокам
- Capture thread использует `unsafe impl Send` для `PulseCapture` (pa_simple thread-safe на практике)
- TUI thread никогда не блокируется на audio — полная отзывчивость, ~60fps
