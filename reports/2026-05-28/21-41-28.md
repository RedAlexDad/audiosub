# Report 2026-05-28 21:41 UTC

## What was added

- `audiosub.toml` — основной TOML-конфиг в корне проекта с секциями `[audio]`, `[asr]`, `[subtitle]`
- `reports/` — директория для отчётов к коммитам

## What was changed

- `src/subtitle/buffer.rs` — добавлено `max_duration_ms`: `SubtitleBuffer` теперь режет длинные сегменты Vosk (20+ сек) на части не длиннее `max_duration_ms`, равномерно распределяя слова
- `src/config.rs`:
  - `SubtitleConfig.max_duration_ms: u64` (default: 10000)
  - `Config::load()` — ищет `audiosub.toml` сначала в CWD, потом в `~/.cache/audiosub/`, потом defaults
- `src/cli.rs` — убраны все `env = "AUDIOSUB_*"` (кроме `AUDIOSUB_CONFIG`); добавлен `--max-duration`
- `src/main.rs`:
  - `resolve_model_path()` — CLI > TOML > default, путь резолвится в absolute (Vosk требует absolute)
  - `max_duration` пробрасывается в `SubtitleBuffer::new()` для обоих режимов (CLI и TUI)
- `.env` — сжат до одной строки (device override)
- `.env.example` — убран, только комментарий о TOML-конфиге
- `Makefile` — убраны `include .env` / `export` (env-переменные больше не читаются приложением)

## Problems encountered

- `resolve_model_path()` не собирался из-за `&PathBuf` вместо `&Path` (clippy `ptr_arg`)
- Clippy ругался на `.map(|p| p.clone())` вместо `.cloned()`
- `String::display()` не существует — модель теперь `String`, не `PathBuf`

## How they were solved

- Тип параметра изменён на `&Path` через auto-deref; `.map(|p| p.clone())` → `.cloned()`
- `.display()` заменён на прямую вставку `{}` для `String`
