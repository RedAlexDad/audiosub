# Report 2026-05-30 00:10 UTC

## What was added
- `lib/.gitkeep` — каталог `lib/` в репозитории для Docker build context
- Dockerfile: `ARG ENGINE` (vosk/whisper/both), мультистейдж, trixie-slim
- Makefile: `ENGINE ?= vosk`, цели `docker-build`/`docker-run` с параметром
- `docker-compose.yml`: build args, ENGINE env, volume для моделей
- `lib/vosk/` в `.gitignore`

## What was changed
- Dockerfile: полная переработка — `ARG ENGINE`, условная установка Vosk (local .so или curl fallback), `/runtime-libs/` для COPY между стейджами, runtime переключён на `debian:trixie-slim` (GLIBC 2.41), добавлены clang/cmake/rustfmt для whisper-rs-sys
- Dockerfile: `RUSTFLAGS="-L /usr/local/lib"` для поиска libvosk.so при LLD-линковке
- Makefile: `docker-build` копирует host `libvosk.so` в `lib/vosk/` перед сборкой, `docker-run` передаёт ENGINE в compose
- `docker-compose.yml`: параметризован ENGINE, models volume вместо встраивания модели в образ

## Problems encountered
- GLIBC mismatch: `rust:latest` (GLIBC 2.41) vs `debian:bookworm-slim` (GLIBC 2.36)
- Whisper-rs-sys требует clang (bindgen), cmake (C++ сборка), rustfmt (формат bindgen)
- LLD-линковка не видит `-lvosk` при `ENGINE=both`, несмотря на `RUSTFLAGS="-L /usr/local/lib"` (флаг есть в выводе линкера, но LLD не находит библиотеку)
- Vosk SDK не имеет SONAME, что усложняет ldconfig

## How they were solved
- Runtime переключён на `debian:trixie-slim` (GLIBC 2.41)
- В builder добавлены `clang`, `cmake`, `rustup component add rustfmt`
- `libvosk.so` копируется и в `/usr/local/lib/` и в `/usr/lib/x86_64-linux-gnu/`
- Для CI добавлен curl-fallback загрузки Vosk SDK, для локальной сборки — COPY из `lib/vosk/` (через Makefile)
- `ENGINE=both` (LLD + -lvosk) остаётся нерешённым
