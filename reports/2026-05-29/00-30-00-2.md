# Report 2026-05-29 00:30:00 UTC

## What was added
- `src/asr/whisper_backend.rs` — реальная реализация WhisperEngine:
  - Скользящее окно аудио (max 30 сек)
  - Периодический запуск инференса (каждые ~3 сек нового аудио)
  - Дедупликация сегментов по consumed_ms
  - trim буфера после обработанных сегментов
  - Конвертация наносекундных таймстемпов whisper в абсолютные ms
- `Makefile`: модель ggml-base.bin, таргеты build-whisper / release-whisper / run-whisper / cli-whisper / model-whisper

## What was changed
- `Cargo.toml`: явно указан `features = ["default"]` для whisper-rs
- `src/main.rs`:
  - create_engine() переделана в рантайм-диспатч по строке из `cfg.asr.engine`
  - Добавлен `#[cfg(feature = "whisper")]` рукав в match
- `Makefile`:
  - Разделены переменные Vosk / Whisper
  - Перечислены все новые таргеты в .PHONY
  - Добавлена секция Whisper в help

## Problems encountered
- `set_max_segment_length` не существует в whisper-rs v0.14; правильное имя — `set_max_len`
- Сборка `whisper-rs-sys` падает с `gnu/stubs-32.h: not found` из-за бага в bindgen; обход через `WHISPER_DONT_GENERATE_BINDINGS=1`
- После `text` перемещён в Segment, повторное использование для `self.partial` требовало clone()

## How they were solved
- Заменён вызов на `params.set_max_len(100)`
- Установлена переменная окружения `WHISPER_DONT_GENERATE_BINDINGS=1` во всех Makefile-таргетах для whisper
- `text.clone()` для Segment, оригинальный `text` — для partial
